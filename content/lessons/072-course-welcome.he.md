:::explanation title="ברוכים הבאים ל-HagPy"
אולי כבר יש לכם רעיון לאתר, לכלי קטן שיחסוך זמן או למוצר שאנשים יוכלו להשתמש בו. עד עכשיו הרעיון היה בעיקר בראש. במסלול הזה תהפכו אותו לקוד עובד שאפשר לפתוח בדפדפן ולשתף עם אחרים.

אין צורך בניסיון קודם. נתחיל מהתקנת הכלים, נסביר כל מושג בשפה פשוטה ונבנה את הידע בהדרגה. בכל שלב תבינו מה אתם עושים, למה עושים זאת ואיך יודעים שהצלחתם.
:::

:::explanation title="למה דווקא Python?"
Python מאפשרת לכתוב קוד ברור ולהגיע במהירות מרעיון לתוצאה. במקום להיאבק בתחביר מסובך כבר בהתחלה, אפשר להתמקד בבעיה שרוצים לפתור ובדרך שבה המוצר צריך לעבוד.

עם Python אפשר לבנות שרתים ואתרי Backend, לעבד נתונים, לבצע אוטומציה, להתחבר למסדי נתונים ולשלב שירותי בינה מלאכותית. זו שפה ידידותית למתחילים, אך היא משמשת גם מערכות אמיתיות בסביבות Production.
:::

:::checklist title="בתוך שבוע תוכלו להעלות מוצר ראשון לאוויר"
- להכין סביבת פיתוח מסודרת במחשב
- ליצור פרויקט Python ולנהל את הספריות שלו עם uv
- לכתוב קוד שמקבל מידע, מעבד אותו ומחזיר תוצאה
- לבנות גרסה ראשונה של אתר או מוצר משלכם
- לשמור את הקוד ב-Git וב-GitHub
- לפרסם את הפרויקט באינטרנט
- לשלוח קישור עובד לאדם אחר
:::

:::explanation title="איך לומדים כאן?"
כל נושא מתחיל בהסבר פשוט של הרעיון ושל הסיבה שהוא קיים. לאחר מכן נראה דוגמה, נפרק אותה לחלקים ונריץ אותה בעצמנו.

לא תסתפקו בהעתקת קוד. תשנו אותו, תבדקו מה קרה ותוסיפו יכולת קטנה משלכם. כך כל שיעור מתקדם מהבנה לפעולה, וכל פעולה מקרבת אתכם למוצר עובד.
:::

:::checklist title="מהרעיון לפרסום בשבעה ימים"
- יום 1: מתקינים את הכלים ומכינים סביבת עבודה
- יום 2: יוצרים פרויקט מסודר וכותבים קוד Python ראשון
- יום 3: מוסיפים למוצר נתונים, תנאים ופעולות
- יום 4: בונים ממשק שאפשר לראות ולהפעיל בדפדפן
- יום 5: מחברים בין חלקי המוצר
- יום 6: בודקים, מתקנים ומשפרים את חוויית השימוש
- יום 7: מפרסמים את המוצר ושולחים קישור עובד
:::

:::explanation title="לא רק לגרום לקוד לעבוד"
במהלך המסלול תלמדו גם כיצד מפתחים מקצועיים חושבים: איך מחלקים בעיה לחלקים קטנים, איך נותנים שמות ברורים, איך מארגנים פרויקט ואיך מאתרים שגיאה בלי להיבהל ממנה.

בהמשך תוסיפו בדיקות אוטומטיות, בדיקות איכות קוד, API עם FastAPI, מסד נתונים ותהליך פרסום מסודר. כל כלי יתווסף כאשר יהיה ברור איזו בעיה הוא פותר.
:::

:::exercise title="המשימה הראשונה שלכם"
כתבו במשפט אחד איזה אתר, כלי או מוצר הייתם רוצים להעלות לאינטרנט. אין צורך לדעת עדיין כיצד לבנות אותו.

שמרו את המשפט. במהלך המסלול תחזרו אליו, תחלקו אותו ליכולות קטנות ותראו כיצד רעיון כללי הופך בהדרגה למוצר עובד.
:::

:::explanation title="מוכנים להתחיל"
אתם לא צריכים לזכור הכול מראש. אתם צריכים להתחיל, לנסות, לטעות, לקרוא את התוצאה ולהמשיך לצעד הבא.

בשיעור הבא נכין את סביבת העבודה שבה תכתבו, תריצו ותפרסמו את הפרויקט הראשון שלכם.
:::
