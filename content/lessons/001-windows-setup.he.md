:::explanation title="בשיעור הזה נלמד"
בסיום השיעור תדעו לפתוח PowerShell, להריץ בו פקודה, לקרוא את התוצאה ולבדוק באיזו תיקייה אתם עובדים.

נבדוק שהמחשב מריץ Windows 10 או Windows 11, ש-PowerShell פועל, שיש חיבור מאובטח לאינטרנט ושאפשר ליצור תיקיות באזור האישי שלכם.

את Git, ‏uv ו-Python נתקין בשיעורים הבאים. בשיעור הזה רק מכינים בסיס נקי ובודקים שהמחשב מוכן.
:::

:::explanation title="מהו PowerShell"
PowerShell הוא כלי שמאפשר לתת ל-Windows הוראות כתובות. הוראה כזו נקראת פקודה.

לרוב, פקודה מורכבת משם שמתאר פעולה ולעיתים מפרטים נוספים שמשנים את אופן הפעולה. לאחר שמריצים פקודה, PowerShell מציג פלט או הודעת שגיאה ואז מחזיר את שורת ההקלדה.

חלון שבו מקלידים פקודות נקרא Terminal. ‏PowerShell הוא הכלי שמפרש ומבצע את הפקודות שבתוך החלון.

אין צורך לפתוח PowerShell בהרשאות מנהל. עבודה רגילה ללא הרשאות מוגברות בטוחה יותר ומספיקה לפעולות שנבצע במסלול.
:::

:::commandStep title="בדקו איזו גרסת Windows מותקנת"
הפקודה winver מבקשת מ-Windows לפתוח חלון מידע המציג את מהדורת המערכת, הגרסה ומספר ה-Build.

פתחו את תפריט Start, כתבו PowerShell ופתחו חלון PowerShell רגיל, לא כמנהלים. לאחר מכן הריצו:

```powershell
winver
```

איך יודעים שזה הצליח? נפתח חלון בשם About Windows ומופיע בו Windows 10 או Windows 11. סגרו את החלון לאחר הבדיקה.
:::

:::commandStep title="בדקו ש-PowerShell פועל"
$PSVersionTable הוא משתנה מובנה שמכיל מידע על סביבת PowerShell הנוכחית. המאפיין .PSVersion בוחר את פרטי הגרסה בלבד.

```powershell
$PSVersionTable.PSVersion
```

איך יודעים שזה הצליח? מופיעים פרטי גרסה ולאחריהם חוזרת שורת ההקלדה. הערכים עשויים להיות שונים בין מחשבים, וזה תקין.
:::

:::explanation title="הכירו את תיקיית המשתמש"
לכל חשבון Windows יש תיקייה אישית שבה אפשר לשמור קבצים ללא הרשאות מנהל.

PowerShell מחזיק את הנתיב לתיקייה הזו במשתנה המובנה $HOME. שימוש במשתנה מאפשר לכתוב פקודה שמתאימה לכל משתמש, בלי לדעת מראש את שם החשבון שלו.
:::

:::commandStep title="הציגו את תיקיית המשתמש"
הפקודה הבאה אינה משנה קבצים. היא רק מבקשת מ-PowerShell להציג את הערך השמור ב-$HOME.

```powershell
$HOME
```

איך יודעים שזה הצליח? מודפס נתיב שמצביע לתיקיית המשתמש שלכם, בדרך כלל בצורה C:\Users\שם-המשתמש.
:::

:::commandStep title="צרו תיקיית עבודה למסלול"
הפקודה New-Item יוצרת פריט חדש. האפשרות -ItemType Directory מגדירה שהפריט יהיה תיקייה, ו--Path קובעת היכן היא תיווצר.

האפשרות -Force מאפשרת להריץ את הפקודה שוב אם התיקייה כבר קיימת. היא אינה מוחקת קבצים שכבר נמצאים בתוכה.

```powershell
New-Item -ItemType Directory -Path "$HOME\HagPy" -Force
```

איך יודעים שזה הצליח? PowerShell מציג פריט בשם HagPy, והתיקייה מופיעה בתוך תיקיית המשתמש שלכם.
:::

:::explanation title="מהי התיקייה הנוכחית"
לכל חלון PowerShell יש תיקייה נוכחית. פקודות שעוסקות בקבצים פועלות בדרך כלל ביחס לתיקייה הזו.

מעבר לתיקייה אינו מוחק או מעביר אותה. הוא רק אומר ל-PowerShell: מכאן אני רוצה לעבוד עכשיו.
:::

:::commandStep title="עברו לתיקיית HagPy"
Set-Location משנה את התיקייה הנוכחית לנתיב שנמסר לה.

```powershell
Set-Location "$HOME\HagPy"
```

איך יודעים שזה הצליח? הפקודה מסתיימת ללא הודעת שגיאה ושורת ההקלדה חוזרת.
:::

:::commandStep title="ודאו שאתם במקום הנכון"
הפקודה Get-Location מציגה את התיקייה הנוכחית. זו פקודת בדיקה בלבד: היא אינה משנה דבר במחשב.

```powershell
Get-Location
```

איך יודעים שזה הצליח? הנתיב שמודפס מסתיים ב-\HagPy.
:::

:::explanation title="בדקו את החיבור הנדרש להמשך"
בשיעורים הבאים נוריד כלים ונעבוד עם שירותים באינטרנט. החיבורים האלו משתמשים ב-HTTPS, שפורט ברירת המחדל שלו הוא 443.

Test-NetConnection בודקת אם המחשב יכול ליצור חיבור ליעד מסוים. הבדיקה אינה מתקינה דבר ואינה שולחת סיסמה או מידע אישי.
:::

:::commandStep title="בדקו חיבור מאובטח לאינטרנט"
הפקודה הבאה מנסה ליצור חיבור ל-GitHub דרך פורט HTTPS.

```powershell
Test-NetConnection github.com -Port 443
```

איך יודעים שזה הצליח? בשורה TcpTestSucceeded מופיע הערך True. אם מופיע False, בדקו את החיבור לאינטרנט, VPN, ‏Firewall או הגבלות של רשת ארגונית לפני שתמשיכו.
:::

:::explanation title="למה ההכנה הזו חשובה"
כאשר פקודה נכשלת, חשוב לדעת אם הבעיה נמצאת בכלי עצמו או בסביבה שסביבו.

הבדיקות שביצענו מפרידות בין המצבים: אנחנו יודעים איזו מערכת הפעלה פועלת, ש-PowerShell מקבל פקודות, שאפשר לכתוב בתיקיית המשתמש ושהמחשב יכול להתחבר לשירות הדרוש להמשך.

כך, אם התקנה בשיעור הבא לא תצליח, יהיה לנו בסיס ברור לאבחון במקום לנחש.
:::

:::warning title="טעויות נפוצות"
PowerShell נפתח כמנהל: סגרו אותו ופתחו חלון רגיל. אין צורך בהרשאות מנהל בשיעור הזה.

הודבקה פקודה אך היא לא רצה: לחצו Enter לאחר ההדבקה.

הנתיב אינו מסתיים ב-HagPy: הריצו שוב Set-Location "$HOME\HagPy" ואז Get-Location.

הקלדתם נתיב וקיבלתם שגיאה: נתיב אינו פקודה. כדי לעבור לתיקייה השתמשו ב-Set-Location.

TcpTestSucceeded הוא False: אל תשנו הגדרות אבטחה באופן אקראי. בדקו קודם שהאינטרנט מחובר, נסו ללא VPN או פנו למנהל הרשת אם זה מחשב ארגוני.
:::

:::exercise title="תרגול קצר"
צרו בתוך תיקיית HagPy תיקייה בשם practice, עברו אליה ובדקו את המיקום. לפני כל פקודה, נסו לומר לעצמכם מה היא אמורה לעשות:

```powershell
New-Item -ItemType Directory -Path practice -Force
Set-Location practice
Get-Location
```

התרגול הצליח אם הנתיב שמודפס מסתיים ב-\HagPy\practice.

לאחר מכן חזרו לתיקיית HagPy. שתי הנקודות .. מציינות את תיקיית האב:

```powershell
Set-Location ..
Get-Location
```

סיימתם כאשר הנתיב שוב מסתיים ב-\HagPy.
:::

:::summary title="סיכום"
- PowerShell מקבל פקודות ומבקש מ-Windows לבצע אותן.
- אין צורך בהרשאות מנהל לעבודה הרגילה במסלול.
- $HOME מצביע לתיקיית המשתמש הנוכחי.
- התיקייה הנוכחית קובעת היכן פקודות רבות פועלות.
- Set-Location עוברת לתיקייה ו-Get-Location מציגה את המיקום.
- Test-NetConnection מאפשרת לבדוק אם ניתן ליצור חיבור ליעד ברשת.
- המחשב מוכן לשיעורי ההתקנה כאשר כל בדיקות ההצלחה בשיעור עברו.
:::

:::terms title="מושגים שלמדנו"
- Terminal: חלון שבו מקלידים פקודות ורואים את הפלט שלהן.
- PowerShell: כלי שמפרש ומריץ פקודות ב-Windows.
- Command: הוראה כתובה לביצוע פעולה מוגדרת.
- Output: המידע שפקודה מציגה לאחר הרצתה.
- Path: כתובת של קובץ או תיקייה.
- Current directory: התיקייה שבה PowerShell עובד כרגע.
- Automatic variable: משתנה ש-PowerShell יוצר ומעדכן בעצמו, כמו $HOME.
- HTTPS: חיבור מוצפן המשמש אתרים ושירותים באינטרנט.
- Port 443: פורט ברירת המחדל לחיבורי HTTPS.
:::
